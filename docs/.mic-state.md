**Brief: Task E — Weekly Resurface Digest**

**Repo:** `lego651/drop-note` (monorepo, pnpm workspaces)
**Base branch:** `main`
**New branch:** `s7/weekly-digest`
**Commit prefix:** `[s7]` (this is Sprint 7 — new feature, not a bug)

---

**Context:**

drop-note is an email-to-dashboard content saver (AGPL-3.0). Users email anything to `drop@dropnote.me`; AI summarizes and tags it; users browse a dashboard. The app is live at `https://dropnote.me`.

Architecture as of D11 (2026-05-26): synchronous AI in `/api/ingest` Vercel function. No BullMQ, no Railway, no worker process. Scheduled tasks must use **Vercel Cron**, not BullMQ.

Weekly Digest is a product-differentiating feature: every Monday, users get an email with (a) this week's top saves and (b) 2 resurface picks from 30+ days ago — the "rediscovery" hook that makes the product sticky.

---

**What to build:**

**1. DB migration**

File: `supabase/migrations/<timestamp>_add_digest_enabled.sql`

```sql
ALTER TABLE users
  ADD COLUMN IF NOT EXISTS digest_enabled boolean NOT NULL DEFAULT true;
```

After adding migration file, note that `pnpm gen:types` will need to be run to regenerate `packages/shared/src/database.types.ts`. Run it as part of this task.

**2. Vercel Cron route**

File: `apps/web/app/api/cron/digest/route.ts`

- Method: `GET`
- Auth: check `Authorization` header = `Bearer ${process.env.CRON_SECRET}`. Return 401 if mismatch.
- Logic:
  1. Query all users where `digest_enabled = true` (use Supabase service role client — `lib/supabase/admin.ts`)
  2. For each user, in parallel (Promise.allSettled):
     a. Query "this week's highlights": items created in last 7 days, status = 'done', ordered by created_at desc, limit 5. If 0 items → skip this user (no email).
     b. Query "resurface picks": items created 30+ days ago, status = 'done', NOT pinned, ordered by RANDOM(), limit 2. If 0 → omit resurface section.
  3. Render email using React Email template (see below)
  4. Send via Resend SDK (already in project at `apps/web/lib/email.ts`)
  5. Log: `console.log('[cron/digest] sent', { userId, weekItems: n, resurface: m })`
  6. Return `{ ok: true, processed: n, skipped: m }` as JSON

**3. Vercel Cron config**

File: `apps/web/vercel.json` (create if not exists, or add to existing crons array)

```json
{
  "crons": [
    {
      "path": "/api/cron/digest",
      "schedule": "0 9 * * 1"
    }
  ]
}
```

That is: Monday 9:00 AM UTC weekly.

Also add `CRON_SECRET` to `.env.example` with a comment: `# Secret for Vercel Cron auth header — generate with: openssl rand -hex 32`.

Add `CRON_SECRET` to `.env.local` docs comment in `CLAUDE.md` key env vars section.

**4. Email template**

File: `apps/web/lib/emails/weekly-digest.tsx`

Use React Email (already in project — see `apps/web/lib/emails/cap-exceeded.ts` and `save-limit-exceeded.ts` for pattern). Build a simple, clean template:

- Subject line: `Your drop-note week — [N] saves this week`
- Section 1: "This week's saves" — list the week's items (title or subject line, source type badge, link to dashboard)
- Section 2: "From the vault (resurface picks)" — 2 items from 30+ days ago with "Re-read →" link. Omit this section entirely if no resurface items.
- Footer: "You're receiving this because you have weekly digest enabled. [Unsubscribe]" — unsubscribe link = `https://dropnote.me/settings` (just link to settings, no one-click unsubscribe token needed for MVP)
- Style: match existing email style in the other templates (inline CSS, no external fonts)

**5. Settings UI toggle**

File: `apps/web/app/(dashboard)/settings/SettingsClient.tsx`

Add a "Weekly Digest" section with a toggle switch (shadcn `Switch` component):
- Label: "Weekly digest email"
- Description: "Receive a weekly email every Monday with your top saves and resurface picks."
- Toggle state reads from `users.digest_enabled` (Supabase query on mount)
- On toggle: PATCH `users` row via Supabase client, update `digest_enabled`
- Optimistic UI update (update local state immediately, revert on error)
