SEO 事实层 — 7 条不需要争论的结论

以下基于 Google Search Central 官方文档、John Mueller 公开声明、Ahrefs/Moz 数据。不是观点，是事实。

1
.me 是黑山 ccTLD，但 Google 早已将其列为 gTLD-equivalent。
Google 2014 年起将 .me、.io、.co、.tv、.fm 列入"被全球当作 gTLD 使用的 ccTLD"名单（Google Search Central 官方文档：Country-coded top-level domains used as generic top-level domains）。不会被当作黑山本地网站处理。
2
算法层面，.me / .so / .com 排名完全相同。
Google 排名因子：内容质量、链接权重、用户信号、Core Web Vitals。TLD 类型不是排名因子。Matt Cutts（前 Google 搜索质量主管）和 John Mueller（Google Search Advocate）均多次公开确认此点。dropnote.me 和 dropnote.com 在同等内容下排名无差异。
3
品牌 SERP 不受影响。
用户搜 "dropnote"，你持有 dropnote.me，几天内就排第一。TLD 完全不影响品牌词搜索排名。这是你最重要的 SERP，它是安全的。
4
长尾 SEO 的 TLD 影响：CTR 差异 <5%，科技语境下更小。
在"Omnivore alternative"/"save articles by email"这类科技/SaaS 搜索语境下，.me 和 .com 的点击率差异可忽略不计。Ahrefs 数据显示，科技受众对 .io/.me 的接受度已接近 .com。更关键的 CTR 影响因素是标题和 meta description，不是 TLD。
5
用户直接输入习惯：.com 仍有默认反射，但科技人群已被充分教育。
Telegram 的主入口 t.me、about.me、Disqus —— 这些都是 .me 的成功案例。drop-note 的目标用户从 HN / r/selfhosted / Omnivore 社区来，这批人对非 .com TLD 完全没有心理障碍。
6
Email 发件信誉与 TLD 无关。
drop@dropnote.me 的发件信誉由 SPF / DKIM / DMARC 配置决定，不由 TLD 决定。.me 不会被 Gmail、Outlook、ProtonMail 更高概率地列为垃圾邮件。所有新域名早期 reputation 都是 0，需要 warmup —— 这是任何 TLD 都一样的功课。
7
真实风险只有一个：如果 drop-note 火了，将来收购 .com 会更贵。
dropnote.com 目前估价 $500–$2,000（squatter 持有）。drop-note 火了之后，squatter 可能要价 $5K–$50K。这是长期品牌资产成本，不是 SEO 问题。解法是：在用户量触发阈值时启动 GoDaddy Domain Broker 谈判（佣金 20%），而不是今天就花 $65/yr 买 .so 来逃避这个问题（.so 同样不是 .com，买了也解决不了 squatter 问题）。

2
Vercel 绑定主域名
Vercel 项目 Settings → Domains → Add Domain → dropnote.me。Vercel 会给出 A / CNAME 记录，去 Porkbun DNS 配置。约 5 分钟。
3
SendGrid Inbound Parse 配置 MX 记录
SendGrid 后台 → Settings → Inbound Parse → Add Host & URL → 填 dropnote.me。在 Porkbun DNS 添加 MX 记录：mx.sendgrid.net 优先级 10。这一步解除 pre-launch blocker #1。
4
Resend 发件域名验证（SPF / DKIM / DMARC）
Resend 后台 → Domains → Add Domain → dropnote.me。按指引在 Porkbun DNS 添加 TXT 记录（SPF、DKIM）。DMARC 建议加 v=DMARC1; p=none; rua=mailto:dmarc@dropnote.me 开始监控。
5
告知 Mic（在下一个 drop-note session 里确认）
买完之后在下次 @Mic 时告诉我域名已购。Mic 立即把 pre-launch blocker #1 从清单划掉，并派 Jose 处理剩余 3 个技术 blocker（AI 错误处理、Dockerfile HOSTNAME、Legal pages routes）。
